# Copilot Instructions — V V AI Website Redesign

## Overview

You are assisting with the V V AI website redesign project. This project has comprehensive project documentation in `_project docs/` that serves as the single source of truth for all design, copy, component, data, image, and animation decisions.

**Golden Rule:** Before creating new pages, components, or making significant changes, consult the relevant reference files in `_project docs/`. All content and design decisions must align with the documented specifications.

---

## Reference File Decision Tree

### When building a NEW PAGE

1. Check `_project docs/vvai-redesign-plan.md` Section 3 (Sitemap) and Section 4 (Page Sections) for content, copy, and structure.
2. Check `_project docs/page-templates.md` for the exact Next.js file path and component layout.
3. Check `_project docs/component-library.md` for any UI components referenced in the template.
4. Check `_project docs/design-system.md` for CSS tokens and styling.
5. Check `_project docs/image-assets.md` for any images or CDN URLs needed.
6. Check `_project docs/animations.md` for animation requirements.

### When adding/modifying a COMPONENT

1. Check `_project docs/component-library.md` to see if it already exists and what its props/structure should be.
2. Check `_project docs/design-system.md` for all CSS classes, colours, and typography tokens.
3. Check `_project docs/animations.md` if animation is involved.

### When adding COPY or HEADLINES

1. Check `_project docs/vvai-redesign-plan.md` Section 4 (Page Sections) for the exact text — never invent copy.

### When using IMAGES or CDN URLs

1. Check `_project docs/image-assets.md` Sections 2–9 for the correct CDN URLs and asset specifications.
2. Ensure `next.config.ts` includes the correct `remotePatterns` for the CDN domain.

### When adding ANIMATIONS

1. Check `_project docs/animations.md` for the exact implementation code.
2. Use the specified library (GSAP, Framer Motion, Three.js, etc.).
3. Wrap in `shouldReduceAnimation()` guard if the animation is non-essential.

### When adding DATA (modules, suites, testimonials, etc.)

1. Check `_project docs/data-models.md` Section 2 for pre-populated static data arrays.
2. Import and use the existing arrays — do not duplicate data inline.

### When adding TYPE DEFINITIONS

1. Check `_project docs/data-models.md` Section 1 for the TypeScript interfaces.
2. Do not re-define types that already exist.

### When setting up i18n or LOCALIZATION

1. Check `_project docs/data-models.md` Section 3 and Section 6 for i18n routing and message key structure.
2. Check `_project docs/page-templates.md` Section 1 for locale-aware layout setup.

## Documentation Update Requirements

### When to update `_project docs/` files

After making significant changes to the codebase, you **must** update the relevant documentation file to keep it in sync:

#### `vvai-redesign-plan.md`

- **Update Section 4 (Page Sections)** if page content, copy, or structure changes

- **Update Section 14 (Implementation Notes)** if discovered decisions deviate from the plan (e.g., routing changes, component API changes, asset loading strategy)

#### `page-templates.md`

- **Update the relevant template section** if the Next.js file structure or page component order changes

#### `component-library.md`

- **Add or update component sections** when creating new UI components or modifying existing ones
- **Document props, anatomy, and default styles** so future work can reference the component spec

#### `data-models.md`

- **Update data arrays** (SUITES, MODULES, TESTIMONIALS, etc.) if static data changes

#### `design-system.md`

- **Update Tailwind config** in Section 1 if new design tokens or CSS custom properties are added
- **Update colour or typography scale** in Section 3 if changes to the visual system are made
- **Update component CSS patterns** in Section 4 if Button, Card, or other shared styles are refined

#### `image-assets.md`

- **Update CDN URLs** in Sections 2–9 if image URLs or asset paths change
- **Update `next.config.ts` remotePatterns** in Section 12 if new image domains are added

#### `animations.md`

- **Add or update animation code blocks** if new animations are implemented or existing ones are modified
- **Update the animation timeline table** in Section 12 after adding new animations

## Key Constraints & Standards

- **Never invent copy.** If a page needs text, it exists in `_project docs/vvai-redesign-plan.md` Section 4. Use it exactly.
- **Never hardcode styles.** All colours, spacing, typography use tokens from `design-system.md`. Check before writing any CSS.
- **Never duplicate components.** Check `component-library.md` first.
- **Never duplicate data.** Use the arrays in `data-models.md` (SUITES, MODULES, TESTIMONIALS, etc.).
- **Never invent images.** Use the CDN URLs from `image-assets.md`.
- **Never hardcode routes.** Use the routing structure from `page-templates.md` and locale-aware navigation from `data-models.md` Section 3.
- **Dark mode first.** The site uses dark mode by default with light mode as a theme toggle. Semantic token utilities (`bg-bg-base`, `text-text-primary`) ensure automatic adaptation.
- **Performance first.** All animations use GPU-composited properties (`transform`, `opacity`). Heavy animations are lazy-loaded and wrapped in `prefers-reduced-motion` checks.
- **i18n compliance.** Content must support locales: `en`, `zh`, `zh-tw`. Use `next-intl` and the message key structure from `data-models.md` Section 6.
